Open Source 與開放授權的基本概念

關於開放授權與 Open Source 的基本概念，建議可以閱讀這些文章：

  • 創用 CC 是什麼？ / Creative Commons
  • 開源是一種生活態度！六個針對 open source 的大哉問 / Buzzorange

在網路上取用任何素材 ( 影像、源碼函式庫、圖示、樣板包 ) 時，務必確認素材的使用授權。常見的授權包括：

  • Creative Commons ( 創用授權 ), 其中還包含：
    • Share Alike (SA) - 以相同方式分享
    • No-Derivs(ND) - 禁止改作
    • Non-Commercial(NC) - 禁止商業利用
    • Attribution(BY) - 需標註作者姓名

    在使用創用 CC 授權時，常以縮寫組合上述幾種授權方式並附上對應連結使用。範例使用方式：本課程網頁使用 CC-BY-SA 4.0 授權方式釋出。



Github 實戰

本次課程請同學依照上課指示於 Github 建立帳號並新建一個專案 ( Repo )。專案需包含一個分支( Branch )，其名為「gh-pages」。專案 gh-pages 的根目錄至少包含一個「index.html」檔案。

完成後，嘗試開啟「https://<你的帳號名>.github.io/<你的專案名>/ 」，即可瀏覽該專案 gh-pages 分支中的 index.html 檔。例如，本課程網頁實際使用 Github 架設於https://zbryikt.github.io/digital-content-design/。

接著，使用 Github Desktop 將專案庫同步至本地電腦。
